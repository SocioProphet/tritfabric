# Contributing

We accept changes only if they:
1) preserve vendor neutrality (no product-locked assumptions),
2) include measurable acceptance criteria (benchmarks, SLAs, or compatibility rules),
3) do not weaken schema/versioning discipline.

## Process
- Add/modify standards in `docs/standards/`.
- For any non-trivial change, add an ADR in `adr/`.
- Update benchmark workloads if behavior changes.
